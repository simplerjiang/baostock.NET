## 4. 量纲与精度差异

| 维度 | 三源差异 | 处理方案 |
| --- | --- | --- |
| 价格 | **东财用整数 × 10^-`f152`**（默认 `f152=2`），腾讯/新浪是浮点 | `RealtimeQuote.Price` 内部用 `decimal`，东财解析时 `value / 10^f152` |
| 成交量 | **腾讯/东财=手；新浪=股** | 统一对外 `long Volume`（股），新浪直采，腾讯/东财 ×100 |
| 成交额 | 三源都给元，但腾讯多一份"万元"且小数处理不同 | 统一 `decimal Amount`（元） |
| 涨跌幅 | 东财 ×10^-2（185 表示 1.85%），腾讯直接给百分数原值 | 统一 `decimal ChangePercent`（百分数原值） |
| 总市值 | **腾讯 亿元 vs 东财 元**（差 8 个量级） | 统一 `decimal MarketCap`（元） |
| 时间戳 | 腾讯 `yyyyMMddHHmmss`、新浪 `yyyy-MM-dd` + `HH:mm:ss` 拆两段、东财 Unix 秒、东财 5min `yyyy-MM-dd HH:mm`、腾讯 5min `yyyyMMddHHmm`、巨潮 Unix 毫秒 | 统一对外 `DateTimeOffset`（北京时间 +08:00） |
| 财报数值 | 东财数字 / 新浪字符串 | 新浪 `decimal.Parse`；统一 `decimal?` |
| 同比 | 东财 `_YOY` 百分数（15.66 = 15.66%）；新浪 `item_tongbi` 小数（-0.12826 = -12.826%） | 统一"百分数原值"，新浪 ×100 |

## 6. 失败与降级实测

实测全部 45 条最终成功，但首轮失败 8 条（17.8%）：

| 故障 | 次数 | 性质 |
| --- | --- | --- |
| 东财 nginx 502 | 3 | 上游不稳，重试 + 降级 host |
| 本地代理瞬断 | 4 | 客户网络，重试 |
| 腾讯 web3.ifzq SSL 异常 | 2 | DNS 多 host 之一不健康，重试到其它 host |
| 东财 rc=102 (kline) | 2 | 缺 `ut` token |
| 东财 9501 (billboard) | 1 | 缺 `columns=ALL` |
| 东财 `{"$type":"1"}` (财务空) | 2 | `companyType` 行业不匹配 |

**结论**：每条 HTTP 调用至少需要 3 次重试 + 切 host + 切 ut；与 ROADMAP §3.3 Hedged Requests 完全契合，但**单源内部还需指数退避**才能让 hedge 真正"无感"。

## 7. 对 Sprint 1 开发的具体建议

1. **`HttpDataClient` 必须默认带 4 个 header**：`User-Agent`、`Referer`（按源不同）、`Accept-Encoding: gzip, deflate`、巨潮还要 `Content-Type: application/x-www-form-urlencoded`。建议通过 `RequestPreparer` 注入。
2. **`EastMoneyKLineSource` 解析必须先 `string.Split(',')`**，OHLCV 是逗号分隔字符串而非 JSON 数组。建议 `static EastMoneyKLineRow Parse(ReadOnlySpan<char> line)` 用 Span 零分配 split。
3. **`EastMoneyRealtimeSource` 必须读 `f152` 决定价格小数位**，不要硬编码 ÷100；同时对 fields 列表去重（`HashSet<string>`），否则响应里出现重复 key。
4. **`SinaRealtimeSource` 按字符位置 split** `,`，第 30/31 字段拼成一个 `DateTimeOffset`（`yyyy-MM-dd` + `HH:mm:ss`，时区 +08:00）。新浪买卖盘 5 档**顺序是 qty,price**，与腾讯/东财相反。
5. **巨潮 PDF 必须支持 Range 请求**：`Accept-Ranges: bytes` 头存在，`Range: bytes=0-1023` 返回 206 + 1024 字节。`CninfoAnnouncementSource.FetchAsync` 应返回 `Stream`，并支持 `long? resumeOffset` 参数。
6. **东财财务源**根据 `BaostockClient.QueryStockBasic(code).Industry` 映射 `companyType ∈ {1=证券, 2=保险, 3=银行, 4=通用}`。SDK 内置 `IndustryToCompanyTypeMap`。
7. **东财财务**单次只能查一期，**新浪一次返回 10 期**。`GetBalanceSheetsAsync(code, fromDate, toDate)` 内部决定循环次数。
8. **新浪四表的 `item_value` 是字符串**，需 `decimal.Parse(InvariantCulture)`，空值是 JSON `null`。
9. **同比统一**：东财 `_YOY` = 百分数原值；新浪 `item_tongbi` = 小数。SDK 归一化为 `decimal? ChangePercent`，新浪解析时 ×100。
10. **股票代码正反映射**：东财 `secid=1.600519`/`0.000001`/`116.430047`（116=BJ），腾讯/新浪 `sh600519`/`sz000001`/`bj430047`，巨潮 `600519,gssh0600519`，baostock TCP `sh.600519`。`CodeFormatter` 必须加 BJ 分支。
11. **时间戳归一化**：所有源最终对外 `DateTimeOffset (+08:00)`。各 parser 单元测试覆盖：腾讯 `20260424102119`、新浪 `2026-04-24` + `10:21:19`、东财 Unix 秒、东财 5min `2026-04-24 10:25`、腾讯 5min `202604241025`、巨潮 Unix 毫秒。
12. **fixture 入库**：`tools/source-probes/raw/` 45 个文件可放进 `tests/Baostock.NET.Tests/Fixtures/v120/`，但财报/已收盘日 K 线作强 fixture，实时行情仅做 "shape" fixture，不参与数值断言。
13. **东财股东接口 13 个子集**：`EastMoneyTopShareholdersSource.FetchAsync` 返回聚合 DTO `ShareholderResearchPage`，内含 13 个独立 `IReadOnlyList<...>`。
14. **Hedge 排序建议**：实时行情 `Sina(0) → Tencent(1) → EastMoney(2)`；K线 `Tencent qfq(0) → EastMoney(1)`；财务 `EastMoney(0) → Sina(1)`；公告 `Cninfo` 唯一源。
15. **东财 realtime 总市值字段是元**（`f116=1810043891463.15`），**腾讯是亿元**（`18101.57`）。两源 `decimal MarketCap` 必须在 source adapter 里立刻归一化，**不能等到外层比对再换算**。
16. **Sprint 1 开头补北交所样本**（如 BJ430047 / 同心传动）独立验证 `CodeFormatter` 与各源 BJ 分支。
